concierge — how design, code and live stay in sync

plain-language first: you plan the design in claude design, and we want that to end up live and documented without you babysitting it. the catch is claude design is locked behind your login, so nothing i run can reach in and grab it. so we put one small manual hop in the loop — you export the file — and automate everything after that.

the shape of it:

  • you author in claude design (plan, wireframe, iterate there — that's where you like to think)
  • one hop: export the standalone html → drop it in design/exports/ → say "sync"
  • i automate the rest: canonical file, git history, render deploy, docs, diffs

one file-drop per design pass buys you version history + a live url + real docs.

optional automation i can add

  • a folder-watch: watch design/exports/ and auto-sync the moment you drop a file — your side becomes just "drop" (trade-off: a background watcher).
  • a diff view: after each sync, render a side-by-side of what changed vs the previous export.

decisions to talk about

  1. same repo or separate? planning docs + the deployed site share one repo (concierge-wireframe). one source of truth, but the repo is public. ok, or split?
  2. public repo — it's public so render builds it cleanly. want it private (i'd confirm render's github app has access first)?
  3. the folder-watch — want "drop and it just deploys", or keep the explicit "sync" word as the trigger?
  4. is claude design really the home? if you'd rather plan in the repo (house-style .md/.html twins i can pull/push freely), we drop the manual hop entirely — but you lose the design canvas.
